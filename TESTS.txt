Pour lancer un test, ouvrir l'interface à l'aide de l'icône de raccourci
sur le bureau ou dans le menu démarrer.
Sélectionner Fichier > Ouvrir le projet .alt 
puis :

  Tests\Exemple\Projets\anonyme.alt

ou :

    Tests\Exemple\Projets\anonyme2.alt

Les fichiers du projet sont affichés dans les onglets centraux ainsi que 
dans le gestionnaire de projets à droite de l'interface.

Cliquer sur l'icône .csv pour lancer un processus de transformation des
données .xhl en données .csv.

Lorsque le message "Terminé" est apparu dans l'onglet de messages de 
l'interface, appuyer sur l'icône "A" bleue pour lancer l'interface
R-Studio. 

Vérifier que le projet altair.Rproj est bien lancé, avec un fichier altair.R
dans la partie supérieure droite de l'interface RStudio. Sinon, ouvrir le 
projet à partir de l'explorateur de fichiers situé dans le quart inférieur 
droit de l'interface. Il est situé sous Users/Public/altair. 

Si le fichier altair.R n'est toujours pas affiché, aller le sélectionner
de même dans l'explorateur de fichier du quart inférieur droit à 
l'emplacement Users/Public/altair/Tests/Exemple

Puis lancer l'exécution d'un rapport en cliquant sur l'icône en forme de
cahier au milieu du quart supérieur droit de l'interface. Sélectionner le format
voulu. Le fichier sera automatiquement affiché au terme de l'exécution par 
l'application adéquate.

Note : pour accéder aux bases .csv en lien du document .pdf, fermer l'affichage
par défaut sous Sumatra puis rouvrir le fichier altair.pdf généré sous 
Tests\Exemple avec acrobat Reader(R) d'Adobe.
