Edition d'un projet .alt

Pour éditer un projet :

  • soit utiliser un éditeur de texte et de suivre les indications supra ;
  • soit utiliser l'éditeur intégré de projet .alt : Ctrl + E ou menu Editer → Editer le projet courant → faire les modifications → menu Fichier interne à l'éditeur → Enregistrer et Quitter ou autres actions de ce menu.
  • soit modifier les paramètres dans l'interface elle-même puis enregistrer le projet (Ctrl + S ou menu Fichier → Enregistrer) ou utiliser la fonctionnalité d'enregistrement automatique de l'interface (menu Configurer → Configurer l'interface → Sauvegarder le projet .alt automatiquement) ;

Génération automatique du projet par défaut

Un projet par défaut (défaut.alt) est automatiquement généré, en l'absence d'autre projet créé par l'utilisateur, dans l'emplacement défini par tools::userdatadir, autrement dit :

  • dans ~/Dev/altair/DONNEES_XHL pour l'administrateur
  • dans ~/Dev/altair/DONNEES_XHL/login pour les autres utilisateurs.

Le répertoire de génération de défaut.alt est celui dans lequel sont placées les données de paye du gestionnaire de fichiers (partie gauche de l'interface).
Sauf spécification contraire de la constante de compilation DONNEES_XHL, ce répertoire est en général :

  • ~/Dev/altair/Tests/Exemple/Donnees/xhl pour l'administrateur
  • ~/Dev/altair/Tests/Exemple/Donnees/xhl/login pour les autres utilisateurs.

Chargement automatique d'un projet Altaïr dans l'interface graphique

  • Il est possible d'utiliser les fichiers de projet .alt pour lancer directement l'interface graphique, chargée des paramètres et données contenus dans le projet.
    L'utilisateur retrouve ainsi exactement l'état de l'interface et du traitement des données qui était en cours lors d'une utilisation précédente. A cette fin :
    • vérifier que l'ensemble des dépendances de bibliothèques est dans le chemin d'exécution par défaut (sous Linux, LD_LIBRARY_PATH des fichiers .desktop est exporté en variable globale)
    • sous Linux (KDE) : Clic droit sur un fichier .alt → Ouvrir avec → Autres → renseigner le chemin de l'interface Altaïr et cocher Mémoriser l'association...
      Un clic (ou deux, selon les bureaux) sur un fichier .alt permet de charger automatiquement le projet en lançant l'interface graphique.
  • Une autre façon de charger automatiquement le projet en ligne de commande est d'indiquer le chemin du projet en premier argument du chemin de l'interface :
    ~/Dev/altair/Interface_linux/gui/x64$ ./Altair ~/projet.alt
  • Une dernière façon de charger automatiquement un projet est d'utiliser le projet par défaut.
    Il est possible de charger automatiquement le projet par défaut en cochant la case Charger le projet par défaut au lancement du dialogue de configuration.
    Si la case Sauvegarder le projet .alt automatiquement est aussi cochée, le projet par défaut défaut.alt a sauvegardé l'ensemble des paramètres de l'interface décrits dans le tableau des paramètres avant sa dernière fermeture.
    Au lancement suivant, l'état antérieur de l'interface est alors automatiquement restauré.
    Avertissement
    Ce paramétrage peut occasionner des plantages de l'interface si les fichiers de paye présents dans le projet automatiquement sauvegardé à la dernière exécution de l'interface ne sont plus présents sur le disque à l'emplacement indiqué par le projet .alt.
    Dans ce cas, écraser le projet défaut.alt et relancer.

Chargement manuel d'un projet Altaïr dans l'interface graphique

Alternativement, après lancement de l'interface :

  • menu Fichier → Ouvrir le projet .alt ou Ctrl + O
  • si le projet est en fichiers récents (menu Fichier → fichiers numérotés par ordre d'apparition), sélectionner le fichier bascule l'interface sur ce projet récent
  • il est aussi possible de désarchiver un projet archivé (menu Fichier → Désarchiver le projet)
  • pour un nouveau projet : Ctrl + N ou menu Fichier → Nouveau projet.

Génération de la ligne de commande de l'application-noyau lhx

Par défaut la ligne de commande de lhx contient les options suivantes, qui sont automatiquement générées :

  • -m : calcul des maxima du nombre de bulletins de paye par mois et du nombre de lignes de paye par agent par bulletin
  • -d ',' : séparateur décimal virgule
  • -s ';' : séparateur de champs point-virgule
  • -E : Générer l'échelon
  • -rank tools::sharedir + "/rank" : fichier exporté dans ~/.local/share/applications/Altair (tools::sharedir par défaut)
    indiquant l'index de la barre de progression
  • ––cdrom : si depuis un disque optique
  • -D ~/Dev/altair/DONNEES_SORTIE /subdir : répertoire d'exportation des bases si subdir est non vide
  • ––esiret Sirets séparés par blancs : si des Sirets sont excus (barrés) dans l'onglet Siret
  • ––ebudget budgets séparés par blancs : si des budgets sont excus (barrés) dans l'onglet Budget
  • ––eemployeur employeurs séparés par blancs : si des employeurs sont excus (barrés) dans l'onglet Employeur

Elle contient en outre des valeurs qui dépendent de l'état des composants (widgets) de l'interface graphique.
En fonction des valeurs d'un des composants identifiés dans le tableau suivant, un ou plusieurs éléments sont ajoutés à la ligne de commande de lhx.
En miroir, une valeur (ou une série de valeurs) est générée entre la paire de balises XML correspondantes dans le projet .alt.
Par exemple, si la case du dialogue de configuration Limiter la verbosité est cochée, le composant MainWindow::defaultQuietBox génère l'option de ligne de commande -q.
En parallèle, une paire de balises quiet est munie de la valeur positive oui dans le projet XML d'extension .alt :

<quiet>
oui
</quiet>

Si la même case est décochée, l'option -q n'est pas insérée en ligne de commande. La paire de balises du projet .alt est alors renseignée par non :

<quiet>
non
</quiet>
