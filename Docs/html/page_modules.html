L'organisation modulaire du logiciel Altaïr

Les modules du logiciel

Le logiciel comprend trois modules exécutable et un module externe sous forme de bibliotèque statique :

  • une interface graphique utilisateur qui produit une ligne de commande
    Le code source de cette interface est distribué dans le répertoire Interface ;
  • cette interface est développée en utilisant les composants (widgets) fonctionnels du module externe fwidgets_lib, réutilisé pour consolider les fonctionnalités de l'interface graphique et réduire le volume de code propre.
    Le code source de ce module externe est distribué dans le répertoire fwidgets_lib ;
  • une application-noyau nommée lhx, qui :
    • exécute cette ligne de commande ;
    • extrait les données de paye pour produire des bases (bulletins et lignes de paye) au format CSV, équivalentes à celles qui sont exportées par le logiciel Xemelios de la DGFIP ;
    Le code source de cette application-noyau est distribué dans le répertoire lhx.
  • un analyseur de données, constitué d'un ensemble de scripts en langage R, qui sont exécutés :
    • soit directement depuis l'interface graphique ;
    • soit en utilisant l'interface de développement RStudio.
    En entrée, ces scripts réimportent en mémoire les bases de paye précédemment exportées au format CSV.
    En sortie, ils produisent des rapports d'analyse de la paye, aux formats Microsoft Word, Libreoffice Writer ou Adobe PDF.
    Ces analyses portent sur la démographie des structures étudiées, les rémunérations des personnels et le respect du cadre statutaire et réglementaires de la paye.
    Une annexe propose quelques indicateurs d'appréciation de la fiabilité des bases de paye et des tableaux auxiliaires en lien, qui peuvent faciliter l'analyse personnalisée de certains aspects qui ne font pas l'objet d'un traitement automatisé.
    Les rapports d'analyse sont accompagnés d'un dossier documentaire, qui comprend des notices méthodologiques et des documents de référence.
    Ils contiennent des liens hypertextes vers des fichiers auxiliaires au format CSV, qui précisent les analyses statistiques et réglementaires présentées dans le rapport.
    Ces scripts d'extension .R sont distribués, pour certains, dans le répertoire racine et, pour la plupart d'entre eux, dans le répertoire Tests/Exemple.

La documentation des modules

Les modules sont documentés dans le répertoire Docs.
En particulier :

  • Pour les modules interface graphique et application-noyau la documentation Doxygen est générée dans le répertoire Docs/html
  • la documentation PDF et la documentation html du module analyseur est générée dans le répertoire Docs/Scripts_R. Les deux sommaires doivent être lus dans ce même répertoire pour que leurs liens relatifs vers des fichiers auxiliaires puissent s'ouvrir.
  • le guide utilisateurs pour les modules Interface graphique et application-noyau (extraction des données) est le fichier Docs/Guide utilisateurs.pptx
  • un guide utilisateurs relatif au module analyseur (rapports d'analyse) a été produit à l'automne 2017 mais n'a pas été, à ce jour, validé par le CAM ; sa diffusion est suspendue.
    Une version provisoire est enregistrée sous Docs/Guide_Altaïr.docx
  • la documentation méthodologique à caractère statistique relative au calcul des rémunérations est le fichier Docs/méthodologie.pdf
  • le rapprochement entre les requêtes réalisées par l'analyseur de données est actualisé dans le fichier Docs/requetes ALTAIR.xlsx (Auteure : Mme Vitalia le Boudec)
  • le guide d'installation est le fichier Docs/projet/INSTALLATION DU LOGICIEL ALTAÏR.pptx
  • une notice à l'attention des testeurs pour le suivi des bogues est proposée dans le fichier projet/MANTIS-suivi des bogues.pdf Le répertoire Docs contient par ailleurs des documents utiles :
    • soit à la poursuite du développement : volumétrie de codage, guide de développement sous Windows (peut être utile en cas de nécessité)
    • soit à l'interprétation des sorties : documents juridiques et statistiques de référence, copie de la documentation en ligne de la DGFIP sur le modèle de données Xemelios, qui est globalement suivi par le module application-noyau, copie de la convention-cadre nationale de dématérialisation.
