En un mot...

Le présent logiciel produit automatiquement des statistiques et des tests de cohérence et de conformité comptable et réglementaire, à partir de bases de données pouvant être lues par l'application Xemelios (Direction générale des finances publiques, http://xemelios.org).

Cliquer sur l'icône "A" bleue sur le Bureau.

Une interface utilisateur s'affiche qui permet d'exécuter l'extraction des données de paye des fichiers .xhl pour les convertir en fichiers plats .csv

Importer les données

  • Par glisser-déposer depuis le bureau ou tout autre dossier vers l'onglet central au milieu de l'interface.
  • Ou bien en déposant les données dans le répertoire 
         C:\Users\Public\Dev\Altair\scripts\R\Donnees\xhl    [Windows]
            ~/Dev/altair/scripts/R/Donnees/xhl                   [Linux]

L'analyse des lignes de paye en mode heuristique

Le logiciel utilise des motifs de recherche comparables à ceux des systèmes documentaires pour identifier les lignes de paye.

Ce système de recherche ne peut pas être parfait mais donne des résultats satisfaisants dans la majorité des cas.

Pour une approche plus approfondie, plus exhaustive ou plus exacte, il est possible de modifier le comportement du logiciel.

Les recherches sur les libellés de paye s'effectuent alors en mode dit exact, à partir des codes de paye.

Ces codes peuvent être renseignés dans l'onglet Codes du dialogue d'options (Ctrl + T puis cliquer sur l'icône Codes) ou bien en important un fichier de codes au format CSV.

Un exemple de fichier de ce type peut être utilisé. Le séparateur du fichier CSV doit être soit la virgule soit le point-virgule. Les données sont renseignées en colonnes pour les éléments de rémunération indiciaire ou indemnitaire correspondant au libellé des colonnes. Le fichier peut contenir des lignes ou des colonnes vides, mais pas à l'intérieur du tableau de données.

Il est possible de ne pas renseigner toutes les colonnes, mais les noms de colonnes doivent être tous présents dans l'ordre, en minuscules ou majuscules, en respectant l'orthographe.

Ce fichier de codes de paye est automatiquement importé lorsqu'il est placé dans le répertoire d'exportation des données (clé, disque...) si son nom est codes.csv

Sinon il peut être importé en utilisant le dialogue d'importation de l'onglet Codes.

Lancer...

En cliquant sur l'icône en forme de base de données quadrillée CSV dans la barre d'icônes.

Par défaut les rapports d'analyse sont automatiquement produits aux formats pdf, docx et odt. Alternativement, sous linux, il est possible d'utiliser RStudio pour les produire à partir des bases de données extraites.

Le rapport d'exécution et les bases de données sont générés dans le dossier clé R-Altair situé sous : 

     C:\Users\Public\Dev\Altair\scripts\R\Donnees\R-Altaïr    [Windows]
            ~/Dev/altair/scripts/R/Donnees/R-Altair                   [Linux]

Ce dossier peut être ouvert en cliquant sur l'icône "dossier" de la barre d'outils ou dans le menu :

           Lancer - Ouvrir le répertoire des bases

Interface RStudio (sous Linux, non fourni sous Windows)

  • L'interface Altaïr permet le lancement de l'interface RStudio pour afficher le script en langage R et lancer son exécution.

  • Affichage du script dans l'interface RStudio

    Au lancement de l'interface RStudio, vérifier que l'onglet actif est bien le fichier 'rapport_msword.R' dans la fenêtre supérieure gauche. Sinon, dans la fenêtre inférieure droite, cliquer sur le fichier rapport_msword.R dans scripts/R. Pour les lancements suivants de l'interface RStudio, cette opération de sélection de fichier ne sera plus nécessaire.

  • Production d'un nouveau rapport par exécution du script.

    Appuyer sur l'icône "Source" à la droite du quadrant supérieur gauche de l'interface, en vérifiant que l'onglet actif de ce quadrant est bien le fichier rapport_msword.R. Ce qui précède est également valable pour un rapport pdf avec le fichier rapport_pdf.R, qui doit alors être sélectionné dans le quadrant supérieur gauche. Il est également possible de produire les deux types de rapport en même temps avec le fichier rapport_msword_et_pdf.R

    Le schéma XML

    Ce programme est conçu pour analyser les schémas de paye définis par le site internet de l'application Xémélios

    Définition des éléments

    Rappel : le ou les traitements bruts forment à la rémunération régulière hors indemnités, avantage en nature, supplément familial, rappel de rémunération

    Le traitement principal indiciaire doit être le premier élément mentionné dans la liste

    Codage de la TABLE

    Par souci d'économie, les codes suivants sont définis pour le Type de la ligne de paye

                                             Code XML         Code Base      Signification
                                        --------------------------------------------------  

                                        "TraitBrut"         T           Traitement
                                        "IndemResid"        IR          Indemnité de résidence
                                        "SupFam"            S           Supplément familial
                                        "AvantageNature"    AV          Avantage en nature
                                        "Indemnite"         I           Indemnité
                                        "RemDivers"         A           Rémunérations diverses
                                        "Deduction"         D           Déductions
                                        "Acompte"           AC          Acompte
                                        "Rappel"            R           Rappel
                                        "Retenue"           RE          Retenue
                                        "Cotisation"        C           Cotisation
                                        "Commentaire"       CO          Commentaire
