
                        ALTAIR

            Analyse des lignes de traitement, 
   attributions indemnitaires et rémunérations diverses


Le présent logiciel produit automatiquement des statistiques et des tests
de cohérence et de conformité comptable et réglementaire, à partir de bases
de données issues de l'application Xémélios (Direction générale des finances 
publiques, http://xemelios.org).


Installation
------------

Voir fichier Installation.txt au dialogue suivant.


Interface Altair
----------------

  Cliquer sur l'icône "A" bleue sur le Bureau ou dans le menu Démarrer 
  dans le répertoire Altair.

  Une interface utilisateur s'affiche qui permet d'exécuter l'extraction des
  données de paye des fichiers .xhl pour les convertir en fichiers plats .csv

Interface RStudio
-----------------

  L'interface Altair permet le lancement de l'interface RStudio pour afficher 
  le script  en langage R et lancer son exécution.

  Pour afficher les rapports de démonstration directement parl'interface,
  les logiciels suivants doivent être préinstallés :

    HTML : n'importe quel navigateur enregistré dans le registre comme application
           par défaut ;

    PDF  : Acrobat Reader d'Adobe;
    
    MSWORD : Microsoft Word

2. Affichage du script dans l'interface RStudio

  Au lancement de l'interface RStudio, vérifier que l'onglet actif est bien le fichier
  'altair.R'
  dans la fenêtre supérieure gauche. 
  Sinon, dans la fenêtre inférieure droite, cliquer sur le fichier 

               Altair\Tests\Exemple\altair.R

  Pour les lancements suivants de l'interface RStudio, cette opération de 
  sélection de fichier ne sera plus nécessaire.

3. Production d'un nouveau rapport par exécution du script.

  Appuyer sur l'icône "carnet de notes" à la droite de l'icône 
  "baguette magique" et à la gauche de 'Run', ou cliquer sur le menu
  Code > Source, ou appluyer simultanément sur Ctrl + Majuscule + S

  Choisir le type de rapport (.html, .pdf, ou MS Word) dans le dialogue de
  l'interface graphique.

  Le rapport d'exécution est généré dans le dossier :

               Altair\Tests\Exemple

  Les bases de données sont produites dans le dossier :

               Altair\Tests\Exemple\Bases

Il est possible de visualiser sous Excel ou Wordpad les bases liées par
des liens hypertextes dans les rapports (.pdf, .docx, .html)

--------------------

L'ensemble du code et des données de démonstration est anonymisé.

Composition du paquet distribué
-------------------------------

Le module Interface graphique est distribué sous forme binaire dans les 
répertoires Interface_w32 ou Interface_w64. 

Le premier type de distribution correspond à une compilation sous Windows64 avec
les dépendances logicielles Qt5.3.2 de l'éditeur Digia, le compilateur étant de
type GNU GCC pour plateforme 32 bits et processeur de type Celeron ou monocoeur. 

Le premier type de distribution correspond à une compilation sous Windows64 avec
les dépendances logicielles Qt5.3.2 de l'éditeur Digia, le compilateur étant de
type Microsoft MSVC2013 pour plateforme 64 bits et processeur de type multicoeur
récent. 

Le module noyau LHX, qui effectue la conversion des données au format .xhl vers
les bases de données au format .csv, est distribué sous forme binaire dans les 
répertoires win32 ou win64. 

Le type de distribution  win32 correspond à une compilation du code source 
écrit en langage C/C++ sous Windows64 et la version MINGW64 du compilateur GNU
GCC 4.9.2 du projet Mingw w64 (http://mingw-w64.sourceforge.net). La version 
win32 est donc une compilation de type 64 bits, qui vient en complément de 
l'interface Interface_w32 pour processeur de type Celeron ou monocoeur.

Le type de distribution  win64 correspond à une compilation du code source
écrit en langage C/C++ sous Windows64 et la  même version MINGW64 du compilateur
GNU GCC 4.9.2 mais pour processeur de type multicoeur récent. Elle vient en 
complément de l'interface sous Interface_w64.

Les scripts R sont distribués sous Tests\Exemple
